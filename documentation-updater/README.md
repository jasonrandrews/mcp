# Documentation Updater

`documentation_updater.py` is a planning tool for updating MCP documentation, since there is so much of it.

It treats the `-m/--message` text passed to the script as the canonical description of the documentation change, then:

- uses `gpt-5.5` to generate optional Codex prompt snippets when OpenAI API configuration is available
- writes a report telling you which repos to clone and what to ask Codex to update

## Usage

From the repo root:

```bash
python3 documentation-updater/documentation_updater.py -m "Update Arm MCP Docker configuration docs to use armlimited/arm-mcp:latest"
```

If `-m` is missing or empty, the script exits with an error.

## What It Produces

The report includes:

- the canonical change message supplied with `-m/--message`
- the local file(s) that should be updated in this repo
- the repos that need follow-up work
- clone commands for those repos
- optional Codex prompts generated by `gpt-5.5`
- manual follow-up surfaces such as Docker Hub and the MCP Registry

## Repos Covered

- `ArmDeveloperEcosystem/arm-learning-paths`
- `github/awesome-copilot`
- `arm/arm-mcp-gemini`
- `docker/mcp-registry` under `servers/arm-mcp`

## Requirements

The script requires:

- a non-empty `-m/--message`

Optional Codex prompt generation requires:

- `OPENAI_API_KEY`
- OpenAI base URL env `OPENAI_API_PROXY_URL`

If those credentials are not configured, or if prompt generation fails, the script still writes the report and omits the generated prompts.

## Updating the upstream modelcontextprotocol registry (that many other registries pull from)

https://modelcontextprotocol.io/registry/quickstart for more info

Edit ../mcp-local/server.json with the new version info

```bash
mcp-publisher login github
mcp-publisher publish server.json
```

Check published metadata with

```bash
curl -s "https://registry.modelcontextprotocol.io/v0.1/servers?search=arm/arm-mcp" | jq '.'
```

## Updating the Docker MCP Toolkit registry

https://github.com/docker/mcp-registry.git for more info

In a fork of the repo, edit servers/arm-mcp/server.yaml with new version info.

Create a PR to the upstream repo.
