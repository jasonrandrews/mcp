# Documentation Updater

`documentation_updater.py` is now a planning tool only.

It does **not** clone repos, edit files, create branches, or generate PR metadata.
It always uses the local `../README.md` as the canonical Arm MCP configuration source, then:

- requires a `-m/--message` explaining what changed
- uses `gpt-5.5` to generate optional Codex prompt snippets
- writes a report telling you which repos to clone and what to ask Codex to update

## Usage

From the repo root:

```bash
python3 documentation-updater/documentation_updater.py -m "Updated the Arm MCP Docker configuration to match the local README"
```

With an explicit report path:

```bash
python3 documentation-updater/documentation_updater.py \
  -m "Updated the Arm MCP Docker configuration to match the local README" \
  --report documentation-updater/reports/arm-mcp-update-plan.md
```

If `-m` is missing or empty, the script exits with an error.

## What It Produces

The report includes:

- the canonical JSON/TOML config extracted from `../README.md`
- the local file that should be updated in this repo
- the repos that need follow-up work
- clone commands for those repos
- optional Codex prompts generated by `gpt-5.5`
- manual follow-up surfaces such as `developer.arm.com`, Docker Hub, and the MCP Registry

## Repos Covered

- `ArmDeveloperEcosystem/arm-learning-paths`
- `github/awesome-copilot`
- `arm/arm-mcp-gemini`

## Requirements

The script requires:

- `OPENAI_API_KEY`
- one OpenAI base URL env var such as `OPENAI_BASE_URL` or `OPENAI_API_PROXY_URL`

It uses those credentials only to generate Codex prompt snippets.
